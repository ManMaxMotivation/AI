# Правила работы с репозиторием

## Git-процесс

1. Перед началом согласованной существенной задачи выполнить `git fetch origin`.
2. Если рабочая директория чистая, обновить `main` только fast-forward командой
   `git pull --ff-only origin main`.
3. Выполнить изменения и подходящие проверки. Существенными считаются изменения
   кода, зависимостей, скриптов, конфигурации, документации и правил проекта.
4. После согласования логически завершенного результата создать отдельный
   осмысленный commit в `main`.
5. Отправить commit командой `git push origin main`.
6. Подтвердить публикацию: `git status --short --branch`,
   `git log -1 origin/main` и `git ls-remote --heads origin main` должны
   указывать на опубликованный commit.

Не выполнять pull при незакоммиченных изменениях и не отправлять временные,
секретные или несогласованные результаты. При расхождении веток сначала
согласовать способ разрешения, а не выполнять принудительную отправку.

## Секреты

- Не добавлять токены, пароли, ключи или персональные данные в отслеживаемые
  файлы, историю Git или сообщения коммитов.
- Для GitHub использовать авторизацию GitHub CLI. Не дублировать его токен в
  `.env` или другом файле проекта.
- Новые локальные настройки добавлять в `.env.example` без реальных значений и
  исключать их рабочий вариант через `.gitignore`.
